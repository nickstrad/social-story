# docs/ structure

- `plans/completed/` — implemented plans, numbered `00`-`14`, in build order (`00-overview.md` is the architecture map; the rest are one plan per feature). Read these to understand how an existing subsystem was built.
- `plans/open/` — plans not yet implemented (currently `15-centralized-theme.md`). A plan moves to `completed/` once merged to `main`.

See the root `CLAUDE.md` for the worktree and plan → judge → fix workflow that consumes these plans.
